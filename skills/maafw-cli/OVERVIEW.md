# maafw-cli Skill

## 解决什么问题

黑盒自动化测试和桌面自动化面临一个长期矛盾：

- **纯脚本方案**（坐标点击、图像识别脚本）执行快、结果确定，但 UI 一变就失效，维护成本高。
- **纯 AI 方案**（多模态大模型直接看屏幕操作）泛化能力强，但推理慢、结果不稳定、调用成本高，无法支撑日常回归。

maafw-cli 将两者结合：AI 提供通用的视觉感知和决策能力，MaaFramework 提供稳定的图像识别和设备操控能力。首次操作由 AI 驱动，验证通过后将操作沉淀为 Pipeline JSON，后续回放无需 AI 参与。

这一模式的核心价值在于：**AI 只在需要理解和判断时介入，日常重复执行由确定性的 Pipeline 完成**，兼顾灵活性和可靠性。

## 核心能力

### 视觉感知

提供多种屏幕识别方式，覆盖从文字到图像到颜色的各类场景：

- **OCR 文字识别**：识别屏幕文字并返回位置，支持正则过滤和区域限定
- **模板匹配（TemplateMatch）**：像素级精确匹配，适合固定 UI 元素和图标
- **特征点匹配（FeatureMatch）**：基于 SIFT 等算法，对缩放、旋转、遮挡具有鲁棒性
- **颜色匹配（ColorMatch）**：按 RGB/HSV 范围检测颜色区域
- **神经网络检测（NeuralNetworkDetect）**：支持 YOLOv8/v11 等 ONNX 模型，处理复杂场景
- **自定义识别（Custom）**：通过 Python 回调接入任意识别算法

每次识别自动附带截图，识别结果以编号引用（e1、e2…）返回，可直接用于后续操作。

### 设备操控

支持完整的人机交互操作：

- 点击、长按、滑动、滚动
- 文字输入、按键（支持组合键、按下/松开分离控制）
- 底层触控（touch-down / touch-move / touch-up）
- 应用启停、设备 Shell 命令（ADB）

### Pipeline 自动化

以 JSON 格式定义多步自动化流程。每个节点包含识别条件和操作动作，节点之间通过 next 字段连接，支持：

- 超时控制和执行间隔配置
- 操作前后延迟
- 反向匹配（等待某元素消失）
- 结果排序和索引选择
- 加载、校验、运行、查看节点定义的完整管理

Pipeline 是 AI 沉淀操作经验的载体——AI 首次执行后生成 Pipeline JSON，后续由 Pipeline 独立回放。

### 自定义扩展

当内置能力不足时，可通过 Python 脚本扩展：

- **CustomRecognition**：自定义识别逻辑，接收截图和 ROI，返回匹配区域
- **CustomAction**：自定义操作逻辑，可调用设备底层接口

自定义回调通过 MaaFramework 的 ctypes 机制直接运行，无 IPC 开销，且可在 CLI 和 Pipeline 中使用。

### 跨平台支持

通过统一的命令接口同时支持两类设备：

- **Windows 桌面窗口**（Win32）：通过窗口句柄或标题连接，支持截屏和输入注入
- **Android 设备/模拟器**（ADB）：通过 ADB 地址连接，支持截屏、触控和应用管理

同一套操作命令和 Pipeline 协议适用于两个平台，跨端用例可复用核心逻辑。

## 工作模式

### AI 探索模式

AI 通过"感知 → 决策 → 操作 → 验证"循环驱动设备。每一步由 AI 观察屏幕内容、判断当前状态、选择操作、确认结果。适合首次执行测试用例、处理未知界面、排查操作失败等需要理解和判断的场景。

### Pipeline 回放模式

将已验证的操作序列固化为 Pipeline JSON 文件，脱离 AI 独立执行。执行速度快、结果确定、无 AI 调用开销。适合日常回归测试和重复性操作。回放失败时可唤起 AI 介入排障或重新录制。

## 基本工作流程

1. **发现设备**：按名称或地址筛选可用的 Windows 窗口或 ADB 设备
2. **建立连接**：连接到目标设备，建立操控会话
3. **感知屏幕**：通过 OCR、模板匹配或截图获取当前屏幕状态
4. **执行操作**：根据识别结果点击、输入、滑动等
5. **验证结果**：再次感知屏幕，确认操作是否达到预期效果

支持同时管理多个设备会话，通过命名会话在不同设备间切换。

## 适用场景

- **游戏黑盒测试**：无需接触游戏源码或集成 SDK，从外部模拟玩家操作验证功能
- **UI 回归验证**：检查界面元素是否出现在预期位置，文字和图标是否正确
- **桌面应用自动化**：操控 Windows 窗口完成表单填写、按钮点击、流程操作
- **模拟器批量操作**：连接 Android 模拟器执行重复性任务
- **CI/CD 集成**：Pipeline 可作为自动化测试步骤集成到持续集成流水线中
